# 弹珠台游戏 — 设计规格书

## 1. 项目概述

- **项目名称**：Neon Pinball（霓虹弹珠台）
- **类型**：经典街机弹珠台网页游戏
- **技术栈**：单文件 HTML + Canvas 2D，零依赖
- **目标用户**：休闲玩家，浏览器直接打开即玩

## 2. 视觉规范

### 场景
- 画布尺寸：480×700px（竖版弹珠台比例），居中显示
- 背景色：深黑 `#0a0a0f`
- 扫描线叠加：1px 横线，间隔 3px，透明度 5%，模拟 CRT 效果

### 调色板
| 元素 | 颜色 |
|------|------|
| 背景 | `#0a0a0f` |
| 挡板渐变 | `#8B5CF6` → `#6366F1` |
| 挡板光晕 | `rgba(139, 92, 246, 0.6)` |
| 弹珠 | `#ffffff` + 青色光晕 `#06B6D4` |
| 弹珠拖尾 | 渐变透明，从白到透明 |
| 砖块行1 | `#F472B6`（粉）|
| 砖块行2 | `#FB923C`（橙）|
| 砖块行3 | `#FACC15`（黄）|
| 砖块行4 | `#4ADE80`（绿）|
| 砖块行5 | `#38BDF8`（蓝）|
| 分数文字 | `#4ADE80` 霓虹绿 |
| UI边框 | `rgba(139, 92, 246, 0.3)` |

### 砖块
- 单块尺寸：70×25px，间距 4px
- 每行 6 块（6×70 + 5×4 = 440px，居中）
- 共 5 行（30块/关）
- 样式：圆角 4px + 内发光 `box-shadow inset` + 碰撞时外发光闪烁

## 3. 游戏机制

### 物理系统
- 重力加速度：0.25 px/frame²
- 弹性系数：0.75（挡板）/ 0.8（墙壁）/ 0.5（砖块）
- 弹珠半径：8px
- 挡板尺寸：120×14px，圆角 7px

### 碰撞检测
- 弹珠 vs 挡板：圆-矩形碰撞，命中后根据命中位置偏转角度（左右边缘命中角度更大）
- 弹珠 vs 砖块：圆-矩形碰撞，命中后反弹并触发碎裂
- 弹珠 vs 墙壁：左右上三面反弹，底部掉落触发扣命

### 关卡设计
| 关卡 | 目标分 | 砖块行数 |
|------|--------|---------|
| 1 | 500 | 5行 |
| 2 | 1200 | 5行（更密）|
| 3 | 2500 | 5行（随机空缺）|
| 4 | 5000 | 5行 |
| 5 | 10000 | 5行（稀疏+高弹性）|

### 计分
- 每块砖：10分 × 当前关卡系数（1~5）
- 连锁奖励：连续命中 +5 分叠加

## 4. 游戏状态机

```
menu → playing → levelComplete → playing (next level) → win
                  ↓
              gameOver → menu
```

- **menu**：显示标题 "NEON PINBALL"，按任意键/点击开始
- **playing**：正常游戏循环
- **levelComplete**：当前关卡目标达成，显示 "LEVEL X CLEAR"，2秒后进入下一关
- **gameOver**：3条命用尽，显示 "GAME OVER" + 最终分数，点击重玩
- **win**：5关全过，显示 "YOU WIN" + 最终分数 + 通关动画

## 5. 粒子系统

砖块碎裂时生成 10 个粒子：
- 颜色：与碎裂砖块同色
- 初速度：随机方向，速度 2-5 px/frame
- 生命值：30帧，透明度线性衰减
- 尺寸：3-6px 圆形

## 6. 弹珠拖尾

保存最近 8 帧位置，每帧画一个半透明圆点，半径和透明度从新到旧递减。

## 8. 验收标准

- [ ] 弹珠受重力下落，碰撞反弹物理自然
- [ ] 挡板实时跟随鼠标，响应零延迟
- [ ] 砖块碰撞产生粒子爆炸
- [ ] 弹珠有可见拖尾效果
- [ ] 5关目标分递进，每关独立重置
- [ ] Game Over 和 Win 状态有明确反馈
- [ ] 扫描线 CRT 效果可见
- [ ] 霓虹发光效果贯穿全局
